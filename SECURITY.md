# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.3.x   | :white_check_mark: |
| < 1.3   | :x:                |

We recommend always using the latest version available on the [Releases](https://github.com/modarken/Bitwarden-PowerTools/releases) page.

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

If you discover a security vulnerability, please report it privately:

1. **Preferred**: Use GitHub's [Security Advisories](https://github.com/modarken/Bitwarden-PowerTools/security/advisories/new)
2. **Alternative**: Email details to the repository owner (see GitHub profile)

### What to Include

Please include as much of the following information as possible:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)
- Your contact information for follow-up

### What to Expect

- **Acknowledgment**: Within 48 hours
- **Status Update**: Within 7 days
- **Fix Timeline**: Varies based on severity and complexity

### Safe Harbor

We consider security research conducted in good faith as authorized and will:
- Work with you to understand and resolve the issue
- Credit you in the fix (if desired)
- Not pursue legal action for good faith research

Thank you for helping keep Bitwarden PowerTools and its users safe! 🛡️
